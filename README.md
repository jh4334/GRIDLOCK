# GRIDLOCK

미로형 타워 디펜스. 빈 평지에 타워를 벽처럼 세워 스폰 지점에서 기지까지 이어지는 적의 경로를 비틀고 늘려, 20웨이브를 버텨내면 승리한다. 적은 뚫린 칸으로만 이동(플로우필드)하며, 길을 완전히 막는 배치는 자동으로 거부된다. 처치 보상으로 타워를 증설·업그레이드하며 미로를 강화하는 것이 코어 루프다. Vanilla TypeScript + Canvas 2D로 게임 프레임워크 없이 구현했다(설계 근거: [DESIGN.md](DESIGN.md)).

## 실행 / 빌드

```bash
npm install     # 최초 1회
npm run dev     # 개발 서버(핫 리로드)
npm run build   # 프로덕션 빌드 → dist/
npm run preview # 빌드 산출물 로컬 미리보기
```

## 조작법

| 동작 | 방법 |
|---|---|
| 게임 시작 | 타이틀 화면에서 클릭 또는 Space |
| 타워 설치 | 하단 빌드 메뉴에서 타워 선택 → 칸 클릭 |
| 타워 선택 | 설치된 타워 클릭 |
| 업그레이드 | 타워 선택 후 `U` (또는 정보 패널 버튼) |
| 판매 | 타워 선택 후 `X` (또는 정보 패널 버튼) |
| 설치/선택 취소 | `Esc` |
| 다음 웨이브 | 컨트롤 바의 "다음 웨이브" 버튼 |
| 배속 | 컨트롤 바의 `x1` / `x2` / `x3` |
| 음소거 | `M` |
| 다시 시작 / 타이틀로 | 승패 화면의 버튼 (또는 `R`로 재시작) |

디버그 치트: `G` 골드 +1000 · `N` 웨이브 스킵 · `H` 히트박스 표시 · `D` 플로우필드 표시.

최고 기록(도달 웨이브 + 남은 라이프)은 브라우저 `localStorage`에 저장되어 타이틀·승패 화면에 표시된다. 클리어 기록이 미클리어보다 우선하고, 그다음 도달 웨이브, 마지막으로 남은 라이프로 더 좋은 기록을 판정한다.

## 배포 (GitHub Pages)

`main` 브랜치에 push하면 `.github/workflows/deploy.yml`이 자동으로 빌드 후 GitHub Pages에 배포한다. 최초 1회 저장소 설정에서 **Settings → Pages → Build and deployment → Source를 "GitHub Actions"로** 지정하면 활성화된다. `vite.config.ts`의 `base: './'` 설정 덕분에 하위 경로(`/<repo>/`) 서빙에서도 자산 링크가 깨지지 않는다.
