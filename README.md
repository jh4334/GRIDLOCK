# GRIDLOCK

미로형 타워 디펜스에서 출발해 미니 RTS까지 아우르는 두 모드의 캔버스 게임이다. Vanilla TypeScript + Canvas 2D로 게임 프레임워크 없이 구현했다(설계 근거: [DESIGN.md](DESIGN.md)).

- **디펜스 모드**: 빈 평지에 타워를 벽처럼 세워 스폰 지점에서 기지까지 이어지는 적의 경로를 비틀고 늘려, 20웨이브를 버텨내면 승리한다. 적은 뚫린 칸으로만 이동(플로우필드)하며, 길을 완전히 막는 배치는 자동으로 거부된다. 처치 보상으로 타워를 증설·업그레이드하고, 3레벨 타워는 2택 스페셜 분기로 특화한다. 20웨이브 승리 후에는 **엔드리스 모드**로 무한 진행하며 최고 도달 웨이브를 기록한다. 맵은 **평원**과 **협곡**(바위 벽 지형) 중에 고른다.
- **정복 모드**: 일꾼으로 크리스탈을 채취하고 배럭·차량 공장·포탑을 지어 AI 진영과 교전하는 미니 RTS. AI 난이도는 **쉬움/보통/어려움** 3단계. 적 본진(HQ)을 파괴하면 승리한다.

그래픽: Kenney.nl (CC0) — `public/assets/kenney/`(라이선스: [LICENSE.txt](public/assets/kenney/LICENSE.txt)). 스킨 로드 전에는 내장 벡터 아트로 폴백한다.

## 실행 / 빌드

```bash
npm install       # 최초 1회
npm run dev       # 개발 서버(핫 리로드)
npm run build     # 타입체크 + 프로덕션 빌드 → dist/
npm run preview   # 빌드 산출물 로컬 미리보기
npm run test:e2e  # Playwright 스모크 스위트(빌드 산출물 대상, 두 모드 + 데모 시나리오)
```

## 조작법

### 공통 / 타이틀

| 동작 | 방법 |
|---|---|
| 모드 선택 | 타이틀에서 "디펜스" / "정복" 버튼 |
| 정복 난이도 선택 | 타이틀 정복 버튼 아래 쉬움 / 보통 / 어려움 |
| 디펜스 맵 선택 | 타이틀에서 평원 / 협곡 |
| 게임 시작 | 타이틀 화면에서 클릭 또는 Space |
| 음량 조절 | 컨트롤 바의 슬라이더(0~100), `M`으로 음소거 토글 — 설정은 `localStorage`에 저장·복원 |
| 다시 시작 / 타이틀로 | 승패 화면의 버튼 (또는 `R`로 재시작) |

### 디펜스 모드

| 동작 | 방법 |
|---|---|
| 타워 설치 | 하단 빌드 메뉴에서 타워 선택 → 칸 클릭(골드가 되는 한 연속 설치) |
| 설치 모드 취소 | **우클릭** 또는 `Esc` |
| 타워 선택 | 설치된 타워 클릭 |
| 업그레이드 | 타워 선택 후 `U` (또는 정보 패널 버튼) |
| 스페셜 분기 | 3레벨 타워 선택 시 정보 패널의 스페셜 2택 버튼(선택 저장·전투 반영) |
| 판매 | 타워 선택 후 `X` (또는 정보 패널 버튼) |
| 선택 취소 | `Esc` |
| 다음 웨이브 / 얼리콜 | 컨트롤 바의 "다음 웨이브" 버튼 — 웨이브 진행 중에도 조기 호출 가능(남은 적 기준 보너스 골드) |
| 배속 | 컨트롤 바의 `x1` / `x2` / `x3` |
| 엔드리스 계속 | 20웨이브 승리 오버레이의 "엔드리스 계속" 버튼 |

배럭 타워는 우클릭으로 병사 집결지를 지정한다.

### 정복 모드

| 동작 | 방법 |
|---|---|
| 유닛/일꾼/HQ 선택 | 클릭, 또는 드래그로 박스 선택 |
| 이동 / 채취 / 집결지 명령 | 선택 상태에서 **우클릭** |
| 공격 이동 | 유닛 선택 후 `A` → 목표 클릭(우클릭/`Esc`로 대기 해제) |
| 부대 지정 | `Ctrl`+`1`~`9`로 현재 선택을 부대로 저장 |
| 부대 선택 | `1`~`9`로 해당 부대 재선택 |
| 건설 취소 | 건설 모드에서 우클릭 또는 `Esc` |

디버그 치트: `G` 골드 +1000 · `N` 웨이브 스킵 · `H` 히트박스 표시 · `D` 플로우필드 표시.

## 데이터 / 밸런스

모든 밸런스 수치는 `src/data/*.json`에만 둔다(코드에 매직넘버 금지). `towers.json`(타워·스페셜 분기·업그레이드), `enemies.json`(적 스탯·특수 능력), `waves.json`(웨이브 구성·얼리콜·엔드리스), `conquest.json`(정복 유닛·건물·난이도), `maps.json`(디펜스 맵 바위 배치), `economy.json`(시작 자원·보상).

최고 기록(도달 웨이브 + 남은 라이프, 엔드리스 최고 웨이브 포함)은 브라우저 `localStorage`에 버전 필드와 함께 저장되어 타이틀·승패 화면에 표시된다. 클리어 기록이 미클리어보다 우선하고, 그다음 도달 웨이브, 마지막으로 남은 라이프로 더 좋은 기록을 판정한다.

## 배포 (GitHub Pages)

`main` 브랜치에 push하면 `.github/workflows/deploy.yml`이 자동으로 빌드 후 GitHub Pages에 배포한다. 최초 1회 저장소 설정에서 **Settings → Pages → Build and deployment → Source를 "GitHub Actions"로** 지정하면 활성화된다. `vite.config.ts`의 `base: './'` 설정 덕분에 하위 경로(`/<repo>/`) 서빙에서도 자산 링크가 깨지지 않는다.
