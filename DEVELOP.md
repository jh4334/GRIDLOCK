# GRIDLOCK — 디벨롭 플랜 (루프 엔지니어링용)

> PLAN.md(M0~M13)는 완료. 이 문서는 그 이후의 개선 백로그다.
> **루프 운영 규칙** (매 이터레이션 공통):
> 1. 이 문서에서 **미완료(`[ ]`) 태스크 중 가장 위의 것 1개만** 골라 구현한다. 한 이터레이션 = 한 태스크.
> 2. 구현 → `npm run build` 통과 → **Playwright 스크린샷으로 DoD 직접 확인** → 커밋·푸시 → 체크박스 갱신.
> 3. CLAUDE.md 규칙 엄수: 밸런스 수치는 JSON만, update/render 분리, 파일 300줄 초과 시 분리, 라이브러리 추가 금지.
> 4. 태스크가 모호하면 이 문서의 해당 항목 설명을 스펙으로 삼고, 설명에 없는 결정은 "가장 단순한 쪽"을 택한다.
> 5. 회귀 주의 지점: 플로우필드 재계산(설치/판매/파괴), 모드 간 키 격리, 배속 서브스텝, 에셋 로드 전 벡터 폴백.

## D1. 검증 하네스 (루프의 안전망 — 최우선)

- [x] D1.1 E2E 스모크 테스트 승격: 세션 스크래치패드에서 쓰던 Playwright 검증 패턴을 레포 `tests/e2e/`로 정식화한다.
  - `tests/e2e/smoke.mjs` 하나로: ① 타이틀 렌더 ② 디펜스: 타워 설치→웨이브 1 시작→처치로 골드 증가 확인 ③ 봉쇄 배치 거부 확인 ④ 정복: 진입→일꾼 생산→크리스탈 증가 확인. 각 단계 스크린샷을 `tests/e2e/out/`에 저장(이 폴더는 .gitignore).
  - 실행기: `node tests/e2e/smoke.mjs` (playwright-core는 devDependency로 추가 — 게임 런타임 의존성이 아니므로 CLAUDE.md 라이브러리 금지 규칙과 충돌하지 않음. 커밋 메시지에 근거 명시), Chromium 경로는 환경변수 `PW_CHROMIUM`(기본 `/opt/pw-browsers/chromium`).
  - package.json에 `"test:e2e": "vite preview --port 4173 & node tests/e2e/smoke.mjs"` 형태(포트 충돌 처리 포함, 종료 시 서버 정리).
  - [DoD: `npm run build && npm run test:e2e`가 로컬에서 통과하고 스크린샷 4장 이상 생성]
- [x] D1.2 이후 모든 태스크의 DoD 검증에 D1.1 스모크 + 해당 기능 전용 캡처를 사용하도록 이 문서 상단 규칙에 이미 반영됨 — 이 태스크에서는 스모크에 "정복 승패 도달(치트 없이 x3 방치 패배)" 시나리오를 추가한다. [DoD: 패배 오버레이 스크린샷 자동 캡처]

## D2. 디펜스 모드 QoL & 주스 ②

- [x] D2.1 연속 설치 & 설치 UX: 설치 모드에서 골드가 되는 한 클릭마다 연속 설치(현행 유지 확인) + **우클릭으로 설치 모드 취소** + 설치 불가 사유 토스트(골드 부족/봉쇄/점유 — 캔버스 하단 1줄, 1.5초 페이드). [DoD: 우클릭 취소와 사유 토스트가 보임]
- [x] D2.2 경로 미리보기: 설치 모드에서 고스트 칸을 벽으로 가정한 **예상 경로**(스폰→기지 최단 경로 폴리라인)를 옅게 오버레이. 봉쇄면 경로 없음 표시. BFS 재사용, 매 프레임이 아니라 호버 칸 변경 시에만 계산. (도로 렌더 시스템 재사용: roadPath.ts — computeRoadCells로 예상 경로 조각을 뽑고 renderRoad(…, preview=true)로 회색 반투명 오버레이) [DoD: 고스트 이동에 따라 예상 경로가 갱신됨]
- [x] D2.3 웨이브 프리뷰: "다음 웨이브" 버튼 옆에 다음 웨이브 구성(적 종류 아이콘×수) 표시. waves.json 데이터 기반, 아이콘은 기존 적 스프라이트 축소 재사용. [DoD: 1~20웨이브 프리뷰가 실제 스폰 구성과 일치]
- [x] D2.4 얼리콜 보너스: 웨이브 완료 전에도 "다음 웨이브" 버튼을 활성화(중첩 웨이브)하고, 남은 적이 있는 상태에서 조기 호출 시 보너스 골드(공식·수치는 waves.json에 추가, 예: 남은 초당 1G 상한 20). WaveManager를 다중 진행 큐로 확장하되 완료 판정·승리 조건이 꼬이지 않게 웨이브별 스폰 큐를 독립시킨다. [DoD: 조기 호출 시 보너스 지급, 20웨이브 승리 정상]
- [x] D2.5 주스 ②: 포탑 발사 반동(포신 2px 후퇴 스프링), 적 사망 시 잔해 데칼(tracksSmall 어둡게, 최대 30개 링버퍼, 페이드), 기지 피격 시 화면 가장자리 붉은 비네트 0.3초. 전부 렌더 계층. [DoD: 세 효과가 화면에 보임]
- [x] D2.6 사운드 옵션: 타이틀·인게임에서 음량 조절(0~100 슬라이더, HTML) + 음소거 상태를 localStorage에 저장·복원. 정복 모드에도 기존 효과음 배선 누락분(건설 완료·유닛 사망·웨이브 경보) 추가. [DoD: 새로고침 후 음량 설정 유지, 정복에서 효과음 재생]

## D3. 정복 모드 확장

- [x] D3.1 원거리 유닛 "포격 전차": 배럭과 별개의 생산 건물 "차량 공장"(비용·스탯 conquest.json, 레이아웃에 적 AI용 좌표도 추가). 사거리 140px 투사체 공격, 근접 유닛보다 HP 낮음. 적 AI 빌드오더에도 공장 1개 추가해 대칭 유지. [DoD: 양 진영이 포격 전차를 생산·교전]
- [x] D3.2 중앙 크리스탈 쟁탈 인센티브: 중앙 크리스탈 2칸의 매장량을 크게(개당 800, conquest.json) 하고 본진 필드는 소폭 축소(300) — 확장 유도. 미니맵에 크리스탈 잔량 비율 표시(점 크기 또는 알파). [DoD: 본진 필드 고갈 후 중앙 쟁탈이 자연 발생]
- [x] D3.3 AI 난이도 3단계: 타이틀 정복 버튼 아래 난이도 선택(쉬움/보통/어려움). 난이도별 attackInterval·빌드오더 길이·시작 크리스탈을 conquest.json의 `difficulty.{easy,normal,hard}` 섹션으로. 현행 밸런스 = 보통. [DoD: 세 난이도의 공격 주기 차이가 실측으로 확인됨]
- [x] D3.4 정복 밸런스 재패스 (측정 결과 기준 충족 — 수치 조정 불요: 보통=타이밍 승리 재현·방치 패배, 어려움=동일 전략 실패): D3.1~D3.3 반영 후 자동 매크로(수비 견고화→웨이브 격퇴 직후 결정타)로 승리 재현이 여전히 가능한지 확인하고 JSON만 조정. 기준: 보통 난이도에서 방치=패배, 타이밍 공격=승리, 어려움은 2회 이상 공방 필요. [DoD: 매크로 승리 스크린샷]
