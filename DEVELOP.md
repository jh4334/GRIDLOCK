# GRIDLOCK — 디벨롭 플랜 (루프 엔지니어링용)

> PLAN.md(M0~M13)는 완료. 이 문서는 그 이후의 개선 백로그다.
> **루프 운영 규칙** (매 이터레이션 공통):
> 1. 이 문서에서 **미완료(`[ ]`) 태스크 중 가장 위의 것 1개만** 골라 구현한다. 한 이터레이션 = 한 태스크.
> 2. 구현 → `npm run build` 통과 → **Playwright 스크린샷으로 DoD 직접 확인** → 커밋·푸시 → 체크박스 갱신.
> 3. CLAUDE.md 규칙 엄수: 밸런스 수치는 JSON만, update/render 분리, 파일 300줄 초과 시 분리, 라이브러리 추가 금지.
> 4. 태스크가 모호하면 이 문서의 해당 항목 설명을 스펙으로 삼고, 설명에 없는 결정은 "가장 단순한 쪽"을 택한다.
> 5. 회귀 주의 지점: 플로우필드 재계산(설치/판매/파괴), 모드 간 키 격리, 배속 서브스텝, 에셋 로드 전 벡터 폴백.

## D7. 맵 확장 — 다양성 패스 (2차 루프)

> 배경: 디펜스 2맵(평원/협곡)·정복 1맵뿐이라 단조롭다. 지형 언어를 넓히고(D7.1) → 그 위에 맵을 늘리고(D7.2, D7.4) → 경로 구조 자체를 바꾸고(D7.3) → 무한 다양성(D7.5)으로 마무리한다. 순서 준수 — 뒤 태스크가 앞 태스크의 시스템을 재사용한다.

- [x] D7.1 지형 타입 확장: maps.json 지형을 `rock` 단일에서 3종으로 —
  - `rock`(현행: 통행×·건설×), `water`(통행×·건설×, 시각 구분: 물 타일 — Kenney tileGrass_transition 계열 또는 청색 틴트 타일 + 잔잔한 펄스), `rough`(자갈밭: **통행 가능하되 적 이속 70%**, 건설 가능 — 전략 지형)
  - Grid의 CellState/isWalkable은 rock·water 동일 취급, rough는 Enemy 이동 속도 보정(칸 기반, 슬로우와 곱연산). 도로 렌더는 rough 위에도 정상
  - maps.json 스키마를 `{ rocks: [...] }` → `{ terrain: { rock: [...], water: [...], rough: [...] } }`로 확장(기존 맵 이관)
  - [DoD: 세 지형이 시각·기능적으로 구분되어 동작 (rough 위 적이 느려짐 실측)]

  - [DoD: 5개 맵(기존 2+신규 3) 전부 진입·20웨이브 진행 가능, 도로가 지형을 정확히 우회]
