# GRIDLOCK — 게임 디자인 & 기술 설계

> 프로젝트 요약: 미로 건설형 타워디펜스로 시작해서, **같은 코드베이스 위에** 유닛 생산·선택·명령을 얹어 미니 RTS까지 진화시키는 단계형 프로젝트. M9에서 완성작 1개가 나오고, 그 이후는 전부 보너스 스테이지.

## 1. 레퍼런스 (북극성)

- **Mindustry** — TD+RTS 하이브리드가 상업적으로도 통한다는 증명. 방향성의 북극성
- **Desktop Tower Defense / Gemcraft** — 미로 건설형 TD의 원형 (M1~M9의 모델)
- **Kingdom Rush** — 배럭에서 병사가 나와 길을 막는 시스템 (M10의 모델)
- They Are Billions — TD식 수비 + RTS 경제의 결합 사례 (M12의 분위기)

## 2. 게임 디자인

### 2.1 코어 루프 (TD 단계)

골드로 타워 설치 → 적 웨이브가 미로 통과 시도 → 처치하면 골드 → 타워 증설/업그레이드로 미로 강화 → **20웨이브 생존 시 클리어**. 기지 라이프 0이면 패배.

### 2.2 맵과 미로 규칙

- 그리드 20×14칸, 1칸 = 48px → 캔버스 960×672
- 스폰 지점(좌측 중앙) → 기지(우측 중앙). 초기 맵은 빈 평지
- **타워 = 벽.** 적은 뚫린 칸으로만 이동 (플로우필드 기반)
- **완전 봉쇄 금지**: 설치 직전 BFS 도달성 검사. 길이 완전히 막히는 배치는 거부 + 붉은 표시 피드백

### 2.3 타워 (데이터 주도 — 아래 수치는 밸런싱 출발점, 전부 towers.json에서 관리)

| 타워 | 비용 | 사거리(px) | 공속(회/s) | 공격력 | 특수 |
|---|---|---|---|---|---|
| 애로우 | 50 | 120 | 1.0 | 12 | 기본 단일딜 |
| 캐논 | 90 | 100 | 0.5 | 25 | 반경 40 스플래시 |
| 프로스트 | 70 | 110 | 0.8 | 5 | 2초간 이속 40% 감소 |
| 스나이퍼(M7) | 140 | 280 | 0.33 | 80 | 초장거리 |

- 각 타워 3단계 업그레이드 (비용 = 원가의 80%/120%, 성능 +40%씩)
- 판매 시 누적 투자액의 70% 환급

### 2.4 적 (enemies.json)

| 적 | HP | 이속(px/s) | 보상 | 비고 |
|---|---|---|---|---|
| 러너 | 30 | 90 | 8 | 빠르고 약함 |
| 그런트 | 60 | 60 | 10 | 기준 유닛 |
| 탱커 | 200 | 35 | 25 | 느리고 단단 |
| 스웜 | 15 | 75 | 4 | 12기 몰려나옴 |
| 보스 | 1500 | 30 | 150 | 5웨이브마다 |

- 웨이브 스케일링 출발 공식: `HP = 기본HP × 1.15^(웨이브-1)`
- 1~10웨이브는 waves.json에 수동 정의(조합의 재미), 11웨이브부터 절차 생성

### 2.5 경제

시작 골드 220, 기지 라이프 20. 처치 보상 + 웨이브 클리어 보너스(20 + 웨이브×5). 이자 시스템 없음(단순 유지).

### 2.6 도파민 설계 (M6 주스 패스)

데미지 숫자 팝업, 처치 파티클, 캐논 명중 시 화면 흔들림, 연속 처치 시 사운드 피치 상승, 웨이브 클리어 팡파레, **배속 버튼(x1/x2/x3)**. 그래픽이 도형이어도 이것들만 있으면 손맛이 남.

### 2.7 RTS 진화 단계 디자인 (M10~)

- **M10 배럭**: 병사 3기를 유지하는 타워. 병사는 근접전으로 적을 **블로킹**(교전 중 적 정지). 배럭 클릭 후 우클릭으로 집결지 지정
- **M11 컨트롤**: 좌클릭 단일 선택, 드래그 박스 다중 선택, 우클릭 이동 명령(A* 개별 경로)
- **M12 정복 모드**: 별도 모드 분기. 내 본진 + 적 AI 본진 + 크리스탈 필드. 일꾼 채집 → 배럭/포탑 건설 → 인구 제한 내 병력 생산 → 적 본진 파괴 시 승리. 적 AI는 고정 빌드오더 + 90초마다 공격 웨이브
- **명시적 비목표**: 멀티플레이, 포그오브워, 리플레이

## 3. 기술 스택 & 결정 이유

**Vite + TypeScript + Canvas 2D, 게임 프레임워크 없음**

- 왜 Phaser를 안 쓰나: (1) 루프/렌더링을 직접 짜는 것 자체가 학습 목표 (2) AI가 프레임워크 버전별 API를 헷갈려 존재하지 않는 메서드를 쓰는 사고가 바닐라에선 원천 차단됨 (3) 의존성 0 = 배포·유지보수 단순
- 왜 TS: M10 이후 엔티티 종류가 폭발할 때 타입이 버그를 막아줌
- 렌더링: 정적 맵은 오프스크린 캔버스에 1회 프리렌더, 매 프레임엔 동적 엔티티만
- 그래픽: ~~1차는 도형+색 (프로토타입 미학)~~ → **"NEON GRID — 홀로그램 전장" 아트 적용됨.** 다크 네이비 회로기판 + 네온 글로우(아군 시안/적 마젠타·레드/자원 앰버). 스프라이트는 로드 시 오프스크린 캔버스에 프리렌더(`src/render/`), `setSprite(key, img)`로 실제 PNG 에셋(Kenney 등) 무수정 교체 가능
- 사운드: Kenney 오디오 팩 또는 jsfxr 계열 생성음
- 저장: localStorage (최고 기록, 설정) — 백엔드 없음
- 배포: GitHub Pages 또는 Vercel 정적 배포

## 4. 아키텍처

```
src/
  main.ts              # 부트스트랩, 루프 시작
  core/loop.ts         # requestAnimationFrame + 델타타임(상한 50ms)
  core/input.ts        # 마우스/키보드 → 게임 이벤트
  game/state.ts        # 상태머신: menu | playing | paused | won | lost
  game/economy.ts      # 골드, 라이프
  game/waves.ts        # 웨이브 스포너
  entities/enemy.ts    # + tower.ts, projectile.ts, unit.ts(M10), worker.ts(M12)
  systems/pathfinding.ts  # BFS 도달성 / 플로우필드 / A*(M11)
  systems/combat.ts    # 타겟팅, 명중, 데미지
  ui/hud.ts            # + buildMenu.ts — 캔버스 밖 UI는 HTML/DOM으로
  data/towers.json     # + enemies.json, waves.json
  debug/cheats.ts      # G키 골드+1000, N키 웨이브 스킵, H키 히트박스 표시
```

핵심 패턴:

- 엔티티 = 배열에 담긴 클래스 인스턴스 (`enemies: Enemy[]`). **ECS 도입 금지** — 이 규모엔 과설계
- `update(dt)`와 `render(ctx)` 엄격 분리. 렌더 함수에서 상태 변경 금지
- 밸런스 숫자는 전부 `data/*.json` — 코드에 매직넘버 금지
- 죽은 엔티티는 프레임 끝에 일괄 제거(filter). 성능 문제가 **실제로 생기면** 그때 풀링/공간해시 (스웜 웨이브 프레임 드랍이 신호). 미리 하지 말 것

## 5. 함정 리스트 (착수 전 필독)

1. **풀 RTS 욕심** — 멀티플레이·넷코드는 이 프로젝트의 금지어. 완전히 다른 난이도의 세계
2. **M6 전에 그래픽 욕심** — 도형으로 재미부터 검증. 스킨은 맨 마지막
3. **한 세션에 여러 태스크** — 스파게티 코드 지름길. 한 세션 = 한 태스크 엄수
4. **밸런싱을 코드로** — 수치는 무조건 JSON. "코드 열어서 숫자 수정"이 시작되면 지옥문
5. **플로우필드 재계산 누락** — 타워 설치/판매/파괴 시마다 재계산. 예상 버그 1순위 지점
6. **AI의 자발적 구조 개편** — 리팩토링 제안은 받되 승인 후 진행
